Documentation Architecture

Git Submodule Docs Pattern

Ein standardisiertes Pattern zur Entkopplung und Versionierung von Bibliotheksdokumentationen über Git-Submodule, optimiert für menschliche Entwickler und autonome AI-Agenten.

Entkopplung der Doku

Dokumentation und Code liegen in separaten Repositories. Das spart Speicherplatz und Bandbreite beim Clonen für Konsumenten, die nur die Spezifikationen lesen möchten.

Version-Matching

Das Submodul wird auf einen spezifischen Git-Tag (z. B. v1.0.0) ausgecheckt. Die Dokumentation stimmt somit immer 100% mit der verwendeten Code-Version überein.

AI Agent Ready

AI-Agenten haben im Workspace durch den standardisierten Pfad docs/sdk-docs/ sofortigen Zugriff auf API-Handbücher, ohne erst das Web oder Quellcode parsen zu müssen.

Das Kernprinzip im Detail

Wenn du eine Bibliothek wie my-library erstellst, veröffentlichst du sie in deiner Paketverwaltung (z. B. über Maven/Gradle lokal in mavenLocal()). Parallel erstellst du ein Docs-Repository (z.B. my-library-docs).

Im Bibliotheksprojekt ist dieses Docs-Repository unter docs/releases eingebunden (mit Schreibrechten). Im Consumer-Projekt, welches die Bibliothek konsumiert, wird dasselbe Docs-Repository als read-only Submodule unter docs/sdk-docs/my-library eingebunden und auf den entsprechenden Release-Tag fixiert.

Vorteil für Agenten:

Ein AI-Agent, der den Auftrag hat, ein Consumer-Projekt anzupassen, sieht den standardisierten Ordner docs/sdk-docs/ und liest direkt die Markdown-Handbücher für die genutzte Version. Er baut dadurch korrekte Schnittstellen-Aufrufe ohne ausprobieren zu müssen.

Git-Befehls-Generator

Gib deine Projektdaten ein und erhalte sofort die passenden Git-Befehle.

Phase A: Setup im Library-Repository 
# 1. Neues Doku-Repository über GitHub CLI anlegen
gh repo create my-organization/my-library-docs --public --clone

# 2. In das Haupt-Library-Repository einbinden (Schreibrechte)
git submodule add git@github.com:my-organization/my-library-docs.git docs/releases
Phase B: Einbindung im Consumer-Repository 
# 1. Doku-Submodul lesend hinzufügen
git submodule add git@github.com:my-organization/my-library-docs.git docs/sdk-docs/my-library

# 2. Version taggen und festpinnen
cd docs/sdk-docs/my-library
git fetch --tags
git checkout tags/v1.0.0
cd ../../

# 3. Änderungen in Git erfassen
git add docs/sdk-docs/my-library
git commit -m "chore: pin my-library docs to v1.0.0"

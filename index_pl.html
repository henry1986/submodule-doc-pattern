Git Submodule Docs Pattern

Standaryzowany wzorzec do rozdzielania i wersjonowania dokumentacji bibliotek za pomocą podmodułów Git, zoptymalizowany pod kątem programistów i autonomicznych agentów AI.

Rozdzielenie dokumentacji

Dokumentacja i kod źródłowy znajdują się w osobnych repozytoriach. Oszczędza to miejsce na dysku i przepustowość podczas klonowania dla odbiorców, którzy chcą tylko czytać specyfikacje.

Dopasowanie wersji

Podmoduł jest przełączany na określony znacznik Git (np. v1.0.0). Dokumentacja jest więc zawsze w 100% zgodna z używaną wersją kodu.

Gotowość dla agentów AI

Agenci AI w przestrzeni roboczej uzyskują natychmiastowy dostęp do instrukcji API za pośrednictwem standaryzowanej ścieżki docs/lib-docs/, bez konieczności przeszukiwania sieci czy kodu źródłowego.

Szczegóły zasady głównej

Kiedy tworzysz bibliotekę taką jak my-library, publikujesz ją w swoim menedżerze pakietów (np. lokalnie w mavenLocal() za pomocą Maven/Gradle). Równolegle tworzysz repozytorium dokumentacji (np. my-library-docs).

W projekcie biblioteki to repozytorium dokumentacji jest podłączane pod docs/releases (z uprawnieniami do zapisu). W projekcie konsumenta używającym tej biblioteki, to samo repozytorium jest podłączane jako podmoduł tylko do odczytu pod docs/lib-docs/my-library i przypinane do odpowiedniego znacznika wersji.

Korzyść dla agentów:

Agent AI mający za zadanie zmodyfikować projekt konsumenta wykrywa standaryzowany folder docs/lib-docs/ i bezpośrednio odczytuje instrukcje markdown dla używanej wersji, pisząc poprawne wywołania API bez zgadywania.

1. Utwórz repozytorium docs

Najpierw tworzone jest dedykowane repozytorium GitHub (np. my-library-docs) wyłącznie na podręczniki, przykłady integracji i specyfikacje API. Kod źródłowy pozostaje niezależny.

Generator komend Git

Wprowadź dane swojego projektu, aby natychmiast otrzymać odpowiednie komendy Git.

Fase A: Konfiguracja w repozytorium biblioteki 
# 1. Utwórz nowe repozytorium docs za pomocą GitHub CLI
gh repo create my-organization/my-library-docs --public --clone

# 2. Dodaj jako podmoduł w głównym repozytorium biblioteki (dostęp do zapisu)
git submodule add git@github.com:my-organization/my-library-docs.git docs/releases
Fase B: Integracja w repozytorium konsumenta 
# 1. Dodaj podmoduł dokumentacji z dostępem do odczytu
git submodule add git@github.com:my-organization/my-library-docs.git docs/lib-docs/my-library

# 2. Pobierz i przypnij określony znacznik wersji
cd docs/lib-docs/my-library
git fetch --tags
git checkout tags/v1.0.0
cd ../../

# 3. Zatwierdź (commit) przydzielony podmoduł
git add docs/lib-docs/my-library
git commit -m "chore: pin my-library docs to v1.0.0"
